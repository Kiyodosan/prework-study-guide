Git

  • Enter your Git notes here
  • git status: Checks what branch we are currently on
  • git checkout -b [branch-name]: Creates a new branch and switches to it
  • git add -A: Stages all files in the working branch
  • git commit -m "[add whatever message you want]": Commits all staged files to update your local repo (-m adds a message to the commit, but isn't required)
  • git checkout main: Navigates back to the main branch of your local repo
  • git pull origin main: Loads content from the main branch of your GitHub repo into your local repo. The local main branch and GitHub main branch are in sync when you see the message, "Already up to date." This means that no changes were made to the GitHub main branch while you were working on the feature branch.
  • git push origin [branch-name]: Pushes feature branch to your GitHub repo. Make sure to stage and commit any changes beforehand
